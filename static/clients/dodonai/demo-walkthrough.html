👋 Welcome

This account exists so you never have to juggle local / stage / prod logins mid-demo. It lives on stage and is seeded with five ready-to-show matters, each mapped to a customer persona. Pick the matter that matches the prospect in front of you, follow its click-path, and use the talking points.

🧭 How it works (in one picture)

Every matter runs the same pipeline. Knowing it lets you answer “how does it do that?” with confidence.

Upload → OCR reads every page → Detection splits the bundle into individual records and classifies each → the Rule set sorts, de-dupes, filters and combines → Summarization applies the right template per record type → out comes a chronology + executive summary with clickable citations.

💬 FAQ & objection handling

The first six are the classic framing objections. The rest are the questions prospects actually asked — mined from 94 recorded demo / intro calls (473 capability questions), ordered by how often they came up. Where an answer says “confirm with product,” that capability isn’t verified yet — don’t claim it live.

“Is it just summarizing with ChatGPT?”
No — it detects and splits records, classifies each by type, de-dupes, filters, and applies a structured template per record type, with every line citing its source page. The chronology is auditable, not a black-box paragraph.
“How do I trust the output?”
Every entry links to the exact source page (the PDF REF column). Reviewers verify in one click instead of re-reading the file.
“Our records are messy / mixed types.”
That’s the point — the PI matter starts as a 12-page jumble of ER, imaging, therapy and billing and comes out as 10 clean dated records with the noise filtered.
“Can it match our format?”
Yes — templates are configurable per document type (see the Merit Review and WC matters, which use 5 and 7 templates respectively).
“What about depositions?”
No setup needed — upload the transcript and get both a Page-Line digest and a narrative.
“Is our client data safe in this demo?”
This account is 100% synthetic. For real evaluations we set up a private account.
“How does pricing work — is it just a flat cost per page?” asked on ~14 calls
Pricing is usage-based on the records you put in, not on output length — a 1,000-page bundle that distills to 10 pages is still priced on the input. Basic summaries are a predictable per-page/input cost; advanced and templated summaries cost more because they do more per record. Quote exact rates from the current price sheet — don’t improvise tiers.
“How many pages can it handle? Our records run into the tens of thousands.” ~13 calls
Large mixed bundles are the core use case — upload the whole set (PDF / DOCX / scans); it OCRs every page, splits the bundle into individual records and classifies each. Upload all at once or in batches. For a hard ceiling on a specific tens-of-thousands-of-pages job, confirm the current limit with product.
“Is it HIPAA compliant and SOC-2 certified — and do you train on our data?” ~11 calls
HIPAA-compliant and SOC-2 security posture. The demo uses synthetic data; real evaluations run in a private account so records aren’t co-mingled. For retention, model-training policy, deletion, or a BAA, loop in the team rather than improvising specifics.
“Can it read handwritten, faded, or scanned records?” ~9 calls
Yes — OCR reads every page including handwriting, best-effort: if a person can read it, the system generally can. Scanned PDFs / DOCX are read along with everything else.
“Does it integrate with our case-management / storage tools (Clio, Box, Google Drive)?” ~8 calls
Today it works by direct upload (PDF / DOCX / scans), with records stored securely in Dodonai. For specific integrations (Clio, Box, Drive, API / Zapier), check with product on what exists or can be scoped — don’t promise a connector on the call.
“Can I chat with the documents / ask follow-up questions?” ~8 calls
Don’t claim a chat interface — it isn’t a confirmed feature. What you can show: the chronology, executive summary, and XD extraction answer most “where does X appear / pull every Y” questions, each hyperlinked to its source page. If they want true conversational Q&A, say you’ll confirm with product.
“What can I export to, and can we add our branding?” ~8 calls
Outputs are polished PDF / DOCX reports with hyperlinked source citations; XD extraction exports a cross-record table. Templates control which fields appear per record type. For custom branding (logo / colors / cover page) and Excel-with-editable-notes export, confirm specifics with product.
“Can it filter out billing, fax covers and transmittals so the chronology is clean?” ~8 calls
Yes — those document types can be flagged “skip” so they’re auto-excluded from the chronology and summaries, no manual pre-sorting. (Narrowing a chronology to specific events — e.g. one body part — confirm with product.)
“How long does processing take?” ~5 calls
It depends on set size and complexity — don’t quote a fixed rate; offer a timed eval in a private account. Mechanically it OCRs, splits, classifies, de-dupes, filters and builds the full cited chronology across the whole job.
“Do I build the templates, or do you?” ~5 calls
Either — Dodonai supports configurable per-doc-type templates (one or many, routed by type), and the team can build them with you from your sample docs. Once saved, a template applies consistently. (“Unlimited templates” / duplicate-a-template specifics — confirm with product.)
“Can it pull the same field across many records into one table (all labs, all bills)?” ~5 calls
Yes — cross-document (XD) extraction pulls a field across every record into one exportable table — e.g. every lab value over time, or each provider’s dates and billed amounts — with each doc type routed to its own template.
“If I add records later, do I have to re-run everything?” ~4 calls
New records sort into the chronology in date order and de-duplication collapses repeats across rolling productions. Whether only the new pages reprocess vs a full re-run is an implementation detail — confirm with product before promising it.
“Does it Bates-number records, or do I stamp them first?” ~4 calls
If records carry Bates (or in-house) stamps, it reads the actual stamp — not the facility’s internal pagination — and references it in every citation, so links stay aligned. Unstamped records still get a precise PDF-page citation. Whether the platform applies new Bates stamps to unstamped docs — confirm with product.
“Can I edit or correct the output after it generates?” ~3 calls
You reshape the timeline via detection, multi-template routing, de-dup, filtering and same-date/provider combination, and every entry traces to its source page. For direct in-app editing of a single field, date, or summary line after a run, confirm the exact edit workflow with product.
